                                                                                    Docsify使用时的注意事项

                                                                                    用官网的话来说：一个神奇的文档网站生成器。

                                                                                    我们用别人API或者什么东西的时候都可以看到别人写的一个docs文档，看起来很美观，而我们并不想自己写前端，所以就可以用到这个啦！Docsify！真的很简单很方便。与此同时，也有类似的比如VuePress可以用，这里不做介绍。我在这里只介绍Docsify的注意事项，具体使用看官网去

                                                                                    # 一些注意事项

                                                                                    # 如何进行托管？为什么托管后打不开?

                                                                                    托管，其实直接把docs文件夹里面的东西挂在Github Page或者类似的网页托管服务就可以了，因为本质上并没有使用到npm的包，只是一个网页而已。

                                                                                    如果出现打不开，或者打开非常缓慢的情况，可以把index.html里面所有的外部文件（在VS Code中打开可以看到下划线的）都下载为本地文件，然后引入即可。

                                                                                    # 为什么我托管以后，提示我找不到文件?

                                                                                    这里需要注意一下，如果你的docs可以通过这样的url打开：https://xxx.com，那么应该不会出现这样的问题；但是如果你的docs是这样的url：https://xxx.com/docs，并且你把所有的外部文件都改成了本地文件，那么你需要在你的index.html里面，把所有的本地文件前面都加上一个/docs，例如下面的该法：

                                                                                    <head>
  ...
  <link rel="stylesheet" href="/docs/docsify/vue.css">
  <link rel="icon" href="/docs/favicon.ico">
  ...
</head>
<body>
  ...
  <script src="/docs/docsify/docsify4.js"></script>
  <script src="/docs/docsify/zoom-image.min.js"></script>
  <script src="/docs/docsify/copy-code.min.js"></script>
  ...
</body>

                                                                                    # 为什么使用封面后，点击Start跳转的很生硬？

                                                                                    其实这里是一些前端的知识了，下面是一个封面的例子：

                                                                                    <!-- _coverpage.md -->
 
![logo](_media/icon.svg)
 
# docsify <small>3.5</small>
 
> 一个神奇的文档网站生成器。
 
- 简单、轻便 (压缩后 ~21kB)
- 无需生成 html 文件
- 众多主题
 
[GitHub](https://github.com/docsifyjs/docsify/)
[Get Started](#docsify)

                                                                                    注意到最后面的一个(#docsify)了吗，这代表你需要在你的文档首页（也就是默认的README.mdopen in new window）的第一行，出现一个# docsify，这样就会被自动理解为一个标签，从而进行跳转。如果没有，那么就会非常生硬啦~

                                                                                    如果你说，你不需要第一行出现一个所谓的# docsify，其实也无妨，你可以使用其他的内容，这里只是演示。

                                                                                    # All in all...

                                                                                    其实注意事项还有很多呢，只是我没有遇到，或者暂时没有这方面的需求而已。

                                                                                    不过有什么东西，如果要理解，还是要多尝试，多看看官方的文档，多试一试代码的写法。这样子一旦你试出来了，那么你就是赚到了，以后都学会了；没有试出来，找其他的解决方法拓宽思路，也是不错的选择。
